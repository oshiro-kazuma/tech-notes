# CLAUDE.md でプロジェクトルールをAIに伝える

Claude Codeはリポジトリの`CLAUDE.md`を自動で読んでコンテキストに含める。プロジェクト固有のルールをここに書いておくと毎回説明しなくて済む。

## 書いていること

```markdown
# プロジェクトルール

- PRのタイトルは日本語で書く
- main branchには直接コミットしない
- 新しく作業を始める前に最新のmainからブランチを切る
- ホームディレクトリの絶対パスをコミットされるファイルに含めない

## 設計ドキュメント

詳細な設計方針は `docs/` に置く。関連する実装を変更した場合は
必ずドキュメントの内容と最終更新日も更新すること。

- ローン・返済予定表: `docs/loan-repayment-design.md`
```

## 設計ドキュメントへの参照

`CLAUDE.md`から`docs/`以下の設計ドキュメントを参照させる構成にしている。実装の意図や制約を書いておくと、AIが変な方向に実装するのを防げる。

複雑なドメインロジック（ローンの返済計算など）は特に有効で、「この計算式はこういう理由でこうなっている」を書いておくと壊されにくい。

## 効果

- 毎回「mainに直コミットしないで」と言わなくていい
- ブランチ名の命名規則を守ってくれる
- 設計意図を踏まえた実装をしてくれる
